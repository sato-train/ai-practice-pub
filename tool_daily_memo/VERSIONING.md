# バージョン更新基準

Daily Memo は SemVer をベースに、アルファ版の間は次の基準で更新します。

## 表記

画面下部に `vMAJOR.MINOR.PATCH-alpha · build N` と表示します。

- `MAJOR`: 保存形式・起動方法・互換性に大きく影響する変更
- `MINOR`: 利用者から見える新機能の追加
- `PATCH`: 不具合修正、表示調整、文言修正など既存機能の改善
- `build`: 動作確認済みの配布・保存ポイントごとに 1 ずつ加算

## 更新判断

| 変更内容 | 更新例 | このプロジェクトでの例 |
|---|---:|---|
| 既存のMarkdownやlocalStorageと互換性がない変更 | `1.0.0-alpha` | 保存データ構造の破壊的変更、設定ファイル形式の破壊的変更 |
| 新しい利用機能の追加 | `0.2.0-alpha` | 前日の「明日の優先事項」を翌朝に取り込む |
| 既存機能の修正・UI改善 | `0.2.1-alpha` | 抽出ロジック修正、画面余白調整、ボタン文言修正 |
| 内部整理のみ | バージョン据え置き、必要ならbuildのみ加算 | コメント整理、README補足 |

## 現在の適用

現在の未リリース変更は、翌朝へのインポート機能という利用者向け新機能を含むため、`v0.1.0-alpha` から `v0.2.0-alpha` へ更新します。あわせて確認済みビルドとして `build 3` にします。

## 運用ルール

1. 機能追加・修正を行う。
2. 上の基準で `MAJOR.MINOR.PATCH` を決める。
3. 動作確認できた単位で `build` を 1 つ上げる。
4. 画面表示とこのファイルの「現在の適用」を必要に応じて更新する。
5. コミットメッセージには変更種別を含める。例: `feat: import tomorrow priority into morning memo`
